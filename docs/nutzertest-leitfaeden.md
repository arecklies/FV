### Teil A: Begruessung und Einfuehrung (5 Minuten)

**Vorlesen:**

> Vielen Dank, dass Sie sich die Zeit nehmen, die neue Webanwendung zu testen. Sie kennen die bisherige Software aus Ihrem Arbeitsalltag -- genau diese Erfahrung ist fuer uns besonders wertvoll.
>
> Wir testen heute die Software, nicht Sie. Es gibt keine falschen Antworten oder Handlungen. Wenn etwas unklar oder umstaendlich ist, liegt das an der Software, nicht an Ihnen.
>
> Bitte denken Sie laut: Sagen Sie, was Sie erwarten, wo Sie klicken wuerden und warum. Wenn Sie etwas vermissen, das Sie aus der alten Software kennen, sagen Sie es sofort.
>
> Der Test dauert etwa 60 Minuten. Sie koennen jederzeit Fragen stellen oder abbrechen.

**Abfrage vor Testbeginn:**

| Frage | Antwort |
|-------|---------|
| Seit wann arbeiten Sie mit der bisherigen Software? | _____ Jahre |
| Wie viele Vorgaenge bearbeiten Sie pro Woche? | _____ |
| Welche Aufgaben fuehren Sie am haeufigsten aus? | _____ |
| Welches Bundesland / welche Kommune? | _____ |
| Bildschirmaufloesung am Arbeitsplatz? | _____ |

**Nachfragen:**

**Nachfragen:**
- "In der alten Software: Wie haben Sie Vorgaenge gesucht? Schneller oder langsamer?"
- "Haben Sie die farbigen Statistik-Karten oben als Filter erkannt?"
- "Vermissen Sie einen Suchfilter, den Sie bisher hatten?"
- "Wuerden Sie einen Adress- oder Strassenfilter benoetigen?"

|-----|---------|------|

| 4.2 | Versteht die Ampelfarben (gruen/gelb/orange/rot/dunkelrot)? | Ja / Nein |

**Nachfragen:**

- "Kennen Sie die Moeglichkeit, die Schwellenwerte zu konfigurieren? Wuerden Sie das nutzen?"
- "Wie haben Sie in der alten Software Fristenuebersichten erstellt? War das einfacher oder aufwendiger?"
- "Wuerde Ihnen eine Druckversion des Dashboards helfen (z.B. fuer Besprechungen)?"

#### Block 5: Workflow-Aktionen und Vier-Augen-Freigabe (10 Minuten)

## Qualitative Auswertung -- Kategorisierung

| Kategorie | Schwere | Handlung |
|-----------|---------|----------|
| Blocker | Aufgabe nicht loesbar | Sofort-Fix vor Pilotstart |
| Major | Aufgabe nur mit Hilfe/Umweg loesbar | Fix im naechsten Sprint |
| Minor | Irritation, aber Aufgabe loesbar | Backlog-Item anlegen |
| Wunsch | Nice-to-have, kein aktuelles Problem | Backlog priorisieren |
| Positiv | Explizit gelobt, gut geloest | Beibehalten, kommunizieren |

## Ergebnis-Template

Nach Abschluss beider Testtage:

1. **Executive Summary** (1 Seite): SUS-Scores, NPS, Top-3-Blocker, Top-3-Staerken, Migrationsakzeptanz-Quote
2. **Befundliste** (priorisiert): Jeder Befund mit Kategorie, betroffenes Feature (PROJ-ID), Haeufigkeit, Handlungsempfehlung
3. **Feature-Paritaets-Matrix**: Welche fehlenden Features sind Umstiegs-Blocker (aus Tag 1)?
4. **Onboarding-Empfehlung**: Welche Hilfestellung brauchen Berufseinsteiger (aus Tag 2)?
5. **Backlog-Items**: Jeden Befund der Kategorie Blocker/Major als PROJ-ID via `/po-backlog` anlegen
